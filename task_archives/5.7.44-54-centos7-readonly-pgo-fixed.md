## What Was Fixed

The rebuilt `5.7` flow had split `profile-generate` and `profile-use` across different build roots.

For `5.7`, GCC profile data matching is strongly tied to object paths, so that layout prevented the use-stage build from consuming the generated profile data correctly.

The fix was to restore a single-build-root workflow and add stronger validation around:

- profile/build-root matching
- workload markers in sysbench logs
- explicit runtime identity logging
